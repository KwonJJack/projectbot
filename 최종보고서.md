    중복 노드 제거 및 스레드 세이프 안정화: 1인 개발 단계에서 발생한 server.py 내부의 중복 코드 복사를 제거하고 ros_node.py 모듈 임포트 구조로 일원화. 비동기 웹소켓 루프와 ROS2 타이머 스핀 간의 변수 충돌을 방지하기 위해 threading.Lock() 자원 격리 아키텍처 추가 적용.

4. 핵심 구현 설명 (조원별 담당 파트, 코드 발췌 포함)
4.1 김민성 (팀장 / 통합) - ROS2 노드 안정성 및 하드웨어 락킹 인프라 구축

    설명: 멀티스레드 환경에서 조향 커맨드가 꼬이거나 긴급 정지 상태에서 명령이 강제 실행되는 버그를 막기 위해 threading.Lock()을 바인딩한 스레드 세이프 주행 제어 노드를 설계했습니다.

    코드 발췌 (ros_node.py):

def set_command(self, command):
    command = self.normalize_command(command)
    with self.lock:  # 스레드 상호배제 락 구동
        if self.emergency:
            self.get_logger().warn(f"Command ignored during emergency: {command}")
            return
        self.current_cmd = command
        if command == "forward": self.robot_state = "FORWARD"
        elif command == "backward": self.robot_state = "BACKWARD"
        elif command == "left": self.robot_state = "LEFT"
        elif command == "right": self.robot_state = "RIGHT"
        else: self.robot_state = "IDLE"
    self.get_logger().info(f"CMD : {command}")

async function sendFrameToServer() {
    if (!cameraStream || !isDetecting || isSendingFrame) return;
    isSendingFrame = true;
    try {
        canvas.width = DETECT_IMAGE_WIDTH; // 320px 리사이징으로 추론 병목 제거
        canvas.height = Math.round(webcam.videoHeight * (DETECT_IMAGE_WIDTH / webcam.videoWidth));
        ctx.drawImage(webcam, 0, 0, canvas.width, canvas.height);
        const blob = await new Promise((res) => canvas.toBlob(res, "image/jpeg", 0.55));
        const formData = new FormData();
        formData.append("file", blob, "frame.jpg");
        const response = await fetch("/detect", { method: "POST", body: formData });
        const data = await response.json();
        updateStatus(data);
    } finally { isSendingFrame = false; }
}

    코드 발췌 (server.py):

def decide_follow_command(person_center_x_ratio: float, person_height_ratio: float) -> str:
    left_boundary = 0.40
    right_boundary = 0.60
    too_close_height = 0.62  # 충돌 방지 임계 거리 비율
    
    if person_center_x_ratio < left_boundary: return "left"
    if person_center_x_ratio > right_boundary: return "right"
    if person_height_ratio >= too_close_height: return "stop"
    return "forward"

6. 트러블슈팅 사례 3선 (증상 → 원인 → 해결)
① 모바일 기기 접속 시 마이크 및 카메라 동작 불능 이슈

    증상: 외부망 ngrok 연결을 통해 스마트폰 브라우저로 접속 시, 음성 입력 버튼이나 카메라 시작 버튼을 누르면 Permission Denied 에러 로그가 유발되며 가동 불가능.

    원인: 모바일 웹 표준 보안 정책(W3C 가이드라인) 상 https:// 보안 프로토콜이 보장되지 않는 일반 http:// 터널 접속 환경에서는 마이크/카메라 등 미디어 장치 라이브러리(getUserMedia) 호출이 원천 차단됨.

    해결: 인프라망을 Cloudflare Tunnel로 즉각 이식 및 전환. Quick Tunnel 방식을 통해 유효한 고유 SSL 자격 인증 주소(https://*.trycloudflare.com)를 무료로 발급받아 원격지 스마트폰 디바이스 권한 보안 검증을 완벽히 통과 및 해결.

② 하드웨어 긴급 정지 중 상하좌우 버튼 입력 시 로봇이 다시 발진하는 결함

    증상: 위험 상황이라 판단하여 웹앱에서 Emergency Stop 버튼을 클릭하여 로봇을 멈췄으나, 사용자가 당황하여 수동 조종 패드의 ▲ (전진) 버튼을 클릭하자 터틀봇이 락을 무시하고 앞으로 주행하는 물리적 대형 결함 식별.

    원인: 비동기 웹소켓 스레드로 구동되는 server.py가 명령을 수신하면 ros_node.current_cmd 변수를 즉시 교체하는데, ROS2 타이머 루프가 긴급 정지 플래그(self.emergency)의 유무를 검사하기 전에 변수가 오버라이트되어 조향 토픽이 발행됨. (레이스 컨디션 버그)

    해결: ros_node.py 내부에 from threading import Lock 객체를 생성하고, 변수 대입 연산 영역과 조향 토픽 판단 구절 전체를 with self.lock: 상호배제 구문으로 격리. 긴급 정지 활성화 시 다른 스레드가 명령 변수 자체를 건드리지 못하게 원천 차단하여 안전성 확보.

③ 사람 자율 주행 추적(Follow Mode) 도중 반응 속도가 현저히 느려지며 제어 패킷이 밀리는 현상

    증상: 사람 따라가기 모드를 켜면 스마트폰 화면 인터페이스가 가끔씩 얼어붙고(Freeze), 터틀봇이 사람을 한참 지나쳐 주행하다가 뒤늦게 정지하는 제어 명령 적체 및 딜레이 현상 심화.

    원인: 스마트폰의 기본 비디오 프레임 크기(640x480 이상)와 용량이 보정 없이 그대로 FastAPI 백엔드로 고주기 전송되어 이미지 입출력 병목 현상 발생. 또한 YOLO 연산 스택이 밀리면서 힙 메모리에 대기 중인 패킷 큐가 쌓여 지연 현상이 중첩됨.

    해결: script.js 내부에서 전송 인터벌을 DETECT_INTERVAL_MS = 700으로 넉넉하게 확장 조정하고, 서버로 이미지를 쏘기 전 내부 오프스크린 캔버스를 이용해 해상도를 320px 규격으로 강제 스케일 다운 및 JPEG 품질 55% 압축 적용. 연산량을 대폭 경량화하여 실시간 반응 Latency를 100ms 이내로 제어 완료.
